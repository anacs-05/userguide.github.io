Introdution to GitHub

GitHub is a cloud-based platform designed for storing, sharing, and collaborating on code with others. It hosts Git repositories and offers developers tools to enhance code quality, including command-line functions, issue tracking (for threaded discussions), pull requests, and code reviews.
The benefits of keeping your code in a GitHub "repository" include:

  • Displaying or sharing your work with others.
  • Tracking and managing changes to your code over time.
  • Allowing others to review your code and provide suggestions for improvement.
  • Collaborating on a shared project without affecting your teammates work until you’re ready to merge your changes.

GIT

Collaborative work is one of GitHub’s core features, made possible by the open-source software Git. Git is a version control system that efficiently tracks changes to files, forming the foundation on which GitHub is built. It’s especially valuable when multiple people are working on the same files simultaneously.
Here are some key functions Git enables:

  • Create a branch from the main version of the files that you and your collaborators are working on.
  • Edit files independently and safely within your own branch.
  • Allow Git to intelligently merge your specific changes back into the main version, preventing interference with others' updates.
  • Track changes made by you and others, ensuring everyone works with the latest version of the project.

A version control system (VCS) logs the history of changes as individuals and teams collaborate on projects, allowing you to see what was changed, by whom, and when. In a distributed version control system, each developer maintains a complete copy of the project and its history.

How to Use Git + GitHub


Git and GitHub together form a powerful combination for software development. While they serve different purposes, they complement each other perfectly to support efficient project collaboration and version management.

When you upload files to GitHub, they’re stored in a “Git repository.” This means that as you make changes (or “commits”) to your files, Git automatically begins tracking and managing each modification.

The typical workflow with Git and GitHub involves working on your files locally (on your own computer) and then regularly syncing these changes — along with all related Git history — to a “remote” repository on GitHub.
